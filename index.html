01 · Hakkımda

Piksellerden gerçek dünyaya

Merhaba, ben Esmanur. Gençlik Merkezi Bilişim sınıfında, ekrandaki bir küpü kupaya, bir sinyali oyun mekaniğine, bir kod satırını tarayıcıda gezen bir 3D modele dönüştürmeyi öğreniyorum. Bu sayfa; Blender'da aldığım ilk şekli, dört farklı oyun motorunda kurduğum ilk mekanikleri, Python ve C#'ta yazdığım ilk programları, tarayıcıyı gerçek dünyaya açan WebAR denememi ve VS Code'dan GitHub'a uzanan sürüm kontrolü alışkanlığımı bir araya topladığım kişisel teknik defterim.

Her araç kendi diliyle konuşuyor olsa da altında hep aynı üç işlem yatıyor: bir şeyi kesmek (Blender'da Loop Cut, Python'da slicing), iki parçayı birleştirmek (Bridge Edge Loops, string interpolation) ve gereksiz olanı silmek (Boolean ile delik açmak, .remove() ile listeden çıkarmak). Bu benzerliği fark ettikten sonra yeni bir motor ya da dil öğrenmek, sıfırdan başlamak değil, aynı mantığı yeni bir sözdizimine taşımak oluyor.

Aşağıdaki bölümlerde her araç için hem arayüz/temel kavramları hem de üzerinde çalıştığım ilk projeyi kod/adım adım paylaşıyorum.

“Benim için web, sadece iki boyutlu piksellerden ibaret bir ekran değil; gerçek dünyayla etkileşime giren, yaşayan bir arayüz. Mind-AR, A-Frame ve Three.js kullanarak geliştirdiğim WebAR projelerinde, 3D modelleri ve interaktif içerikleri tarayıcı üzerinden odanıza getiriyorum.” — WebAR projesi motto'su
02 · Bileşenler ve Uygulamalar

Öğrenme yolculuğu

U1

Blender

3D Modelleme

3D Viewport'ta X (kırmızı), Y (yeşil), Z (mavi) eksenleri; G/R/S ile taşı-döndür-ölçekle. Edit Mode'da vertex/edge/face seçimi, Extrude, Inset ve Loop Cut ile topoloji şekillendirme.

İlk Proje

Kahve Kupası Modelleme

Bir silindiri, Edit Mode'da Inset ile duvar kalınlığı bırakıp Extrude ile içini oyarak kupaya dönüştürdüm. Gövdeye iki Loop Cut atıp aradaki yüzeyleri dışarı uzatarak ve Bridge Edge Loops ile birleştirerek kulpu oluşturdum; son olarak Shade Smooth ve Subdivision Surface ile yüzeyi pürüzsüzleştirdim.

1. Küpü sil (X) → Shift+A → Mesh → Cylinder ekle 2. Tab: Edit Mode, üst yüzeyi seç (tuş 3) 3. I (Inset): duvar kalınlığı için içeri pay bırak 4. E (Extrude): aşağı it, kupanın içini oy 5. Ctrl+R: gövdeye iki yatay Loop Cut at 6. Aradaki yüzeyi E ile dışarı uzat (kulp kolu) 7. Uçları X → Faces ile sil, kenarları F ile birleştir (veya sağ tık → Bridge Edge Loops) 8. Tab: Object Mode → sağ tık → Shade Smooth 9. Modifier → Subdivision Surface ekle
U2

Unity

Oyun Motoru

Hierarchy / Inspector / Project üçlüsü, Parent-Child transform ilişkisi. Rigidbody yerçekimini yönetir, Collider katı sınırları belirler; Is Trigger işaretlenince obje katılığını kaybedip bir algılayıcıya dönüşür.

İlk Proje

Altın Toplama Sistemi

Zemine bir Rigidbody eklenmiş küre (oyuncu), üzerine yürüdüğünde toplanan bir küp (altın). Altının Box Collider'ında Is Trigger işaretlenince, katı bir cisim olmaktan çıkıp bir algılayıcıya dönüşüyor; aşağıdaki script bu algılayıcıyı dinliyor.

using UnityEngine; public class CoinCollector : MonoBehaviour { private void OnTriggerEnter(Collider other) { if (other.CompareTag("Player")) { Debug.Log("Altın Toplandı!"); Destroy(gameObject); } } }
U3

Unreal Engine

Oyun Motoru

Sahnedeki her şey bir Actor; içine Static Mesh ve Collision Component gibi bileşenler eklenir. Blueprint, düğümleri (node) birbirine bağlayarak kod yazmadan mantık kurmayı sağlar.

İlk Proje

Zıplama Rampası (Jump Pad)

Box Collision'ın Begin Overlap olayını, Cast To Character üzerinden Launch Character düğümüne bağlayarak, rampaya basan karakteri yukarı fırlatan bir Blueprint kurdum.

1. BP_JumpPad adında bir Actor Blueprint oluştur 2. İçine bir Cube (rampa) ve bir Box Collision (tetikleyici) ekle 3. Box Collision → Details → "On Component Begin Overlap" (+) 4. Event Graph'ta bağla: On Component Begin Overlap → (Other Actor) → Cast To Character Cast To Character (Success) → Launch Character 5. Launch Character → Launch Velocity Z = 1500 6. Compile, Save, sahneye sürükle
Ek Proje — Materyal Editörü

Katmanlı Arazi Materyali

Düğüm tabanlı materyal grafiğinde; yükseklik (Height) ve eğim (Angle/Normal) verilerini Lerp düğümleriyle karşılaştırarak kar, kaya ve çim dokularının birbirine yumuşak geçiş yaptığı bir arazi shader'ı kurdum. Sağdaki panel, materyalin Base Color, Roughness ve Ambient Occlusion çıkışlarını gösteriyor.

Oyun Motoru

İlk Proje

Alan Algılama Mekaniği

Area3D düğümüne bağladığım body_entered sinyali, alana giren nesnenin ismini kontrol ediyor; oyuncu ise hedefe ulaşıldığını yazdırıp queue_free() ile kendini güvenli bir şekilde sahneden siliyor.

extends Area3D func _on_body_entered(body: Node3D) -> void: if body.name == "Oyuncu": print("Hedefe ulaşıldı!") queue_free()
U5

Python

Programlama

Değişkenler (string/int/float), [başlangıç:bitiş] ile slicing, .append() ile veri ekleme, .remove() ile veri silme. Kod yukarıdan aşağıya, satır satır okunur.

İlk Proje

Akıllı Görev Yöneticisi

Bir görev listesi üzerinde dört temel işlemi canlı gösteren konsol uygulaması: .append() ile yeni görev ekleme, [0:3] ile ilk üç görevi dilimleyip acil listesi çıkarma, .remove() ile tamamlanan görevi silme.

gorev_listesi = ["Blender Modeli Yap", "WebAR Kodunu Düzenle", "Unity Sahnesi Kur", "Kitap Oku"] # Yapıştırma (Append) gorev_listesi.append("Python Script Yaz") # Kesme / Dilimleme (Slicing) — 0,1,2. index alınır acil_gorevler = gorev_listesi[0:3] # Delik Açma / Silme (Remove) gorev_listesi.remove("Kitap Oku")

C#

Programlama

Tip güvenli değişkenler (string/int/float/bool), $"..." ile string interpolation, Substring(başlangıç, uzunluk) ile dilimleme. List<T> üzerinde Add, GetRange ve Remove ile veri yönetimi.

İlk Proje

RPG Envanter Sistemi

Tip güvenli bir List<string> üzerinde eşya ekleme (Add), ilk iki eşyayı alt küme olarak kuşanma (GetRange) ve kırılan/kullanılan bir eşyayı silme (Remove) mantığını özetleyen konsol projesi.

List<string> envanter = new List<string> { "Kılıç", "Kalkan", "İksir", "Altın Anahtar" }; // Yapıştırma envanter.Add("Büyülü Asa"); // Kesme — ilk 2 eşyayı kuşan List<string> kusanilanlar = envanter.GetRange(0, 2); // Delik Açma / Silme envanter.Remove("Kalkan");

Proje

Oyun Karakteri Sınıf Hiyerarşisi

Bir Karakter temel sınıfı can ve saldırı gücünü tutuyor; ondan türeyen Dusman sınıfı aynı saldir() metodunu kendi mantığıyla eziyor (override) — polimorfizmi kendi oyun sistemimde deneyimledim.

class Karakter: def __init__(self, isim, can, guc): self.isim = isim self.can = can self.guc = guc def saldir(self, hedef): hedef.can -= self.guc print(f"{self.isim}, {hedef.isim}'e {self.guc} hasar verdi.") class Dusman(Karakter): def saldir(self, hedef): print(f"{self.isim} pusudan saldırıyor!") super().saldir(hedef) oyuncu = Karakter("Esmanur", can=100, guc=15) goblin = Dusman("Goblin", can=40, guc=8) goblin.saldir(oyuncu)
U10

3D Baskı & Ters Mühendislik

Üretim Hattı

Blender'da modellenen ya da taranan bir nesnenin dijital hatalarını (delik, çakışan yüzey) düzeltip 3D yazıcıya hazır hale getirme; ortaya çıkan fiziksel objenin dijital ikizini Unity'de C# ile etkileşimli hale getirme.

Proje

Fiziksel Prototipten Oyun İçi Nesneye

Aynı model, elimdeki fiziksel çıktının "dijital ikizi" olarak Unity'e aktarılıyor; aşağıdaki script, oyuncu bu nesneye çarptığında basit bir fizik tepkisi üretiyor — fiziksel üretim ile kod tabanlı etkileşimi tek bir hatta birleştiren adım burası.

using UnityEngine; public class DijitalIkizEtkilesim : MonoBehaviour { public float itmeGucu = 5f; private void OnCollisionEnter(Collision cerpisme) { Rigidbody rb = cerpisme.rigidbody; if (rb != null) { Vector3 yon = cerpisme.transform.position - transform.position; rb.AddForce(yon.normalized * itmeGucu, ForceMode.Impulse); } } }
